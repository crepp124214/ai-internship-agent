# Security Policy

## Supported Versions

| Version | Supported          |
| ------- | ------------------ |
| 1.x     | :white_check_mark: |

## Reporting a Vulnerability

Do not report security vulnerabilities through public GitHub issues.

Instead, please report them via email to [maintainer email].

Please include the following information:

- Type of issue (e.g., SQL injection, XSS, authentication bypass, etc.)
- Full paths of source file(s) related to the vulnerability
- Location of the affected source code (tag/branch/commit or direct URL)
- Step-by-step instructions to reproduce the issue
- Proof-of-concept or exploit code (if possible)
- Impact of the issue including how an attacker might exploit it

We will aim to respond within 48 hours. If the issue is confirmed, we will
release a patch as soon as possible.
