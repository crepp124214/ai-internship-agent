# 系统架构设计文档

## 1. 系统概述

AI实习求职Agent系统是一个基于多Agent协作的智能求职平台，通过专业化的Agent协同工作，为用户提供全方位的实习求职辅助服务。

## 2. 架构设计原则

- **模块化设计**: 各功能模块独立封装，职责单一
- **可扩展性**: 支持动态添加新的Agent和功能
- **高内聚低耦合**: 核心基础设施与业务逻辑分离
- **配置驱动**: 通过配置文件控制系统行为

## 4. 核心模块说明 (三层架构)

### 4.1 表现层 (Presentation Layer)

**位置**: `src/presentation/`

**职责**：
- 接受用户请求并格式化响应
- 数据验证与转换
- 错误处理与异常传播
- API接口定义

**主要模块**：
- `api/v1/`: REST API接口
- `schemas/`: Pydantic数据验证模型
- `deps.py`: 依赖注入

### 4.2 控制逻辑层 (Business Logic Layer)

**位置**: `src/business_logic/`

**职责**：
- 核心业务逻辑实现
- 任务编排与调度
- 业务规则管理
- Agent协作

**主要模块**：
- `agents/`: 专业化Agent实现
- `services/`: 业务服务（包含核心业务逻辑）

### 4.3 数据持久层 (Data Access Layer)

**位置**: `src/data_access/`

**职责**：
- 数据存储与检索
- 数据库连接管理
- 数据转换
- 事务管理

**主要模块**：
- `entities/`: ORM实体模型
- `repositories/`: 数据访问对象
- `database.py`: 数据库连接与初始化

### 4.4 核心基础设施层 (Core Infrastructure Layer)

**位置**: `src/core/`

**职责**：
- 系统配置管理
- 日志记录
- 异常处理
- 工具函数
- 第三方服务集成

**主要模块**：
- `agent/`: Agent基类与框架
- `llm/`: LLM接口抽象
- `memory/`: 记忆管理
- `tools/`: 通用工具库

## 5. 技术选型

| 层级 | 技术选型 | 说明 |
|------|----------|------|
| Web框架 | FastAPI | 高性能异步API框架 |
| 数据库 | PostgreSQL | 关系型数据存储 |
| 缓存 | Redis | 会话、队列、缓存 |
| 向量库 | ChromaDB | 轻量级向量数据库 |
| ORM | SQLAlchemy 2.0 | Python ORM框架 |
| LLM | OpenAI / Anthropic | 大语言模型 |
| 任务队列 | Celery | 异步任务处理 |
| 测试 | pytest | Python测试框架 |
