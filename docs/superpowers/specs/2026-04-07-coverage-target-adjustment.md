# 覆盖率目标调整

> 日期：2026-04-07 | 状态：草稿

## 问题

当前覆盖率目标 85% 过高，导致：
- `core.tracing` (39.5%) - OpenTelemetry 基础设施，测试环境不加载
- `data_access` (47.4%) - 数据库底层，错误分支难以模拟
- `presentation.api.v1` (51.6%) - API 错误处理路径（500/404/400）

这些都是基础设施代码而非业务逻辑，为"凑覆盖率"写测试没有实际价值。

## 决策

将覆盖率目标从 **85%** 调整为 **80%**。

80% 是 Python 项目的常见标准，覆盖核心业务逻辑，不过度追求异常分支的测试。

## 修改文件

| 文件 | 修改内容 |
|------|----------|
| `pytest.ini` | `cov-fail-under` 从 `78` 改为 `80` |
| `CLAUDE.md` | 测试要求中的覆盖率目标从 `85%` 改为 `80%` |

## 验证

- `python -m pytest tests/unit tests/integration -q` 通过
- 覆盖率 >= 80%

## 影响

当前覆盖率 78.65%，需要提升约 1.4% 才能达到 80% 门槛。
