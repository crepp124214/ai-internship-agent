## 2. 简历页面重构

**文件：** `frontend/src/pages/resume-page.tsx`

### 保留功能
- 创建/导入简历
- 预览摘要 + 保存摘要
- 预览优化建议 + 保存优化建议

### 去掉功能
- 简历列表选择面板（替换为下拉选择）
- 摘要历史面板
- 优化历史面板
- 简历编辑表单（标题/正文/目标岗位）

### 重构后布局

```
┌─────────────────────────────────────────────┐
│ PageHeader: 简历工作台                        │
├──────────────────┬──────────────────────────┤
│ 创建/导入简历     │ 预览区域                  │
│ [标题输入]       │ [简历内容展示]            │
│ [导入文件]       │                          │
│ [创建按钮]       │ [目标岗位输入]            │
│                  │ [预览摘要] [保存摘要]     │
│ 当前简历:        │ [预览优化] [保存优化]     │
│ [下拉选择]       │                          │
└──────────────────┴──────────────────────────┘
```

- 使用 `useQuery` 获取简历列表，通过 Select 下拉切换
- 不再展示完整简历列表
- 去掉简历编辑（保留在 AI 助手中通过对话处理）

## 4. 面试页面重构

**文件：** `frontend/src/pages/interview-page.tsx`

### 保留功能
- 生成题目 + 保存题目
- 面试教练（AI多轮对话）

### 去掉功能
- 创建面试会话
- 题库题目选择器
- 评估回答（预览评分、保存记录、评估记录）

- 去掉创建会话（教练模式内部自动处理）

## 5. 技术实现

### 页面结构
- 每个页面保持单文件组件
- 移除冗余状态（selectedResumeId 等简化为 useQuery 下拉）
- 保留 TanStack Query 用于数据获取

### 组件复用
- 继续使用 `page-primitives.tsx` 中的组件
- `SectionCard`、`PageHeader`、`FormField` 等保持不变
- 按钮统一使用 `PrimaryButton` / `SecondaryButton`

### 路由
- `/tracker` 路由保留在 `App.tsx` 路由配置中，仅删除导航链接
