# Demo Script - 3-5 Minutes Standard Presentation Path

> ⚠️ **演示说明**：以下演示在 Docker 开发环境（mock LLM）下进行。AI 功能（简历优化、面试教练）展示预设响应。数据 CRUD（创建简历、收藏岗位、保存面试记录）真实保存。如需真实 AI 能力，需切换到生产环境并配置有效 API Key。

## 1. 开场：用户痛点与产品定位
- 介绍求职学生面临的主要困境：简历不知道怎么写、面试不知道考什么、岗位信息分散、缺乏反馈机制
- 产品定位：AI 实习求职 Agent，提供简历定制优化、AI 模拟面试、岗位智能匹配和 Agent 助手四大核心功能
- 演示目标：展示从岗位探索 → 简历优化 → 面试准备的完整求职准备流程

## 2. 登录与仪表盘
- 访问 http://localhost:3000 使用演示账号登录（用户名: demo, 密码: demo123）
- 展示仪表盘页面：统计卡片（简历、岗位、面试）和最近活动时间轴
- 解释暖色渐变设计系统：珊瑚#e85d4c + 薰衣草#8b7cf8 的视觉体验
- 指出 Agent 工作区概念：右下角 AI 求职助手按钮和数据导入功能

## 3. 岗位探索与收藏
- 点击“岗位”统计卡片进入岗位探索页
- 展示岗位导入功能：支持 txt、md、json 文件导入，自动解析岗位信息
- 演示手动创建岗位：填写标题、公司、地点、描述等必填字段
- 展示岗位收藏功能：点击“收藏到岗位库”保存岗位到个人岗位库
- 展示岗位列表和预览：显示已收藏的岗位，支持选择岗位查看详情
- 演示岗位匹配预览：选择岗位和简历，点击“预览匹配结果”显示匹配分数和反馈
- 展示一键流转功能：点击“带着岗位去优化简历”无缝跳转到简历优化页面

## 4. 简历优化
- 在简历优化页面展示双栏布局：左侧创建/导入简历和目标设置，右侧简历列表和 AI 功能
- 演示简历创建：填写标题并创建简历，或导入本地简历文件（支持 txt、md、json）
- 展示目标岗位选择：从岗位库选择目标岗位或手动输入目标角色
- 演示 JD 优化指令：可选输入如“突出 Python + FastAPI 项目经历”的定制要求
- 展示浮动工具栏：选中简历文本时出现的优化、摘要、解释悬浮工具
- 演示“一键 JD 定向优化”：点击按钮启动 AI 优化过程
- 展示优化结果四格展示：
  1. 简历摘要：AI 生成的简历核心信息提取
  2. 优化建议：AI 提供的改写和增强建议
  3. JD 定向优化结果：基于目标岗位重写后的简历内容
  4. 匹配报告：关键词覆盖率、匹配分数、 gap 分析和改进建议
- 展示 Diff 视图：Cursor 风格的原文 vs 优化后内容对比，支持接受/拒绝修改
- 展示流程提示：顶部显示“已从岗位探索带入：[岗位名称] / [公司名称]”

## 5. 面试准备
- 从简历优化页面点击面试准备入口或导航到 /interview
- 展示面试准备工作台：左侧题目生成，右侧面试教练对练
- 演示题目生成：导入岗位上下文（可使用优化后的岗位信息），选择简历，设置题目数量，点击“生成题目”
- 展示生成的题目列表：显示题目类型、难度和题目内容预览
- 演示保存到题库：选择题目后点击“保存到题库”按钮
- 展示面试教练对练：点击“开始面试教练”进入多轮对话界面
- 展示 AI 面试流程：
  1. AI 出题并等待回答
  2. 用户输入回答并提交
  3. AI 实时评分并提供反馈和改进建议
  4. 基于回答进行追问（可选）
  5. 循环直到面试结束
- 展示面试结束报告：维度评分（专业知识、思路清晰度、表达能力等）、总体得分和改进建议
- 展示对话历史：完整的问答记录和评分趋势

## 6. 设置中心与 Agent 配置
- 点击用户头像进入设置中心
- 展示设置中心四大卡片网格：
  1. 简历管理：简历列表查看、编辑、删除
  2. 岗位管理：岗位列表查看、编辑、删除
  3. 面试记录：面试历史列表，点击查看详细报告
  4. Agent 配置：LLM 提供商选择和 API 配置
- 展示 Agent 配置页面：
  - Provider 选择器：支持 OpenAI、MiniMax、DeepSeek、智谱 AI 等
  - Model 配置：根据选择的 provider 动态显示可用模型
  - API Key 输入：安全存储用户自带密钥（BYOK）
  - Base URL 配置：可选的自定义端点配置
  - Temperature 参数：调节 AI 创造性和确定性的平衡
- **⚠️ 注意**：在演示模式下，Agent 配置页面用于展示功能界面，实际配置保存和 LLM 调用需要切换到生产环境并配置真实的 API Key。演示模式下 AI 功能使用预设响应。

## 7. 架构亮点与收尾
- 回顾核心技术亮点：
  - Agent 运行时架构：基于 ReAct 模式的统一 Agent 执行框架
  - 多 Agent 协作：简历 Agent、岗位 Agent、面试 Agent 独立运行，各自专注垂直场景
  - LLM 灵活配置：支持多个大模型，可按 Agent 单独配置，实现真正的 BYOK
  - 前后端分离：前端 React + TypeScript，后端 FastAPI + SQLAlchemy，清晰的层次结构
- 强调产品主线：岗位探索 → 简历优化 → 面试准备 的闭环求职准备流程
- 展示架构演进：从伪 Agent 到 Agent Runtime 的重构，砍掉 Tracker 的取舍，为什么收敛成现在这条主流程
