## 背景

Wave 4 要求对外 API 的稳定性和前端 UI polish。在实施过程中发现了多个与 API 契约、安全配置、前端 UI 相关的问题，需要逐一决策和修复。

---

## 决策内容

### D4.1: API 文档补充 — Rate Limiting 章节
- **问题**：`docs/api-reference.md` 缺少速率限制的文档
- **决策**：在 API 文档中新增 Section 7 "Rate Limiting"，包含配置参数、响应格式、环境变量
- **理由**：生产环境必须明确速率限制策略，开发者需要知道 429 响应的处理方式
- **实现**：`docs/api-reference.md` 新增章节

### D4.2: 安全加固 — CORS 生产配置
- **问题**：`docker-compose.yml` 硬编码了 `CORS_ORIGINS=*`，不安全
- **决策**：
  - 确认 `docker-compose.yml` 从 `.env` 继承 `CORS_ORIGINS`（无覆盖）
  - 在 `docs/deployment.md` 中明确要求生产环境必须修改 `CORS_ORIGINS`
  - 确认 Pydantic schema 有输入校验约束
- **理由**：生产环境 CORS 白名单化是基本安全要求
- **实现**：`docs/deployment.md` 新增 section 2.1

### D4.3: 前端 UI — body gradient 修复
- **问题**：`frontend/src/index.css` 中 body 有硬编码渐变色，与设计系统变量不一致
- **决策**：将硬编码渐变 `#f5f1eb → #f0eae2` 改为使用 CSS 变量 `var(--color-canvas)`
- **理由**：保持设计系统一致性
- **实现**：`frontend/src/index.css` body 选择器修改

### D4.4: 测试修复 — 积压测试清理
- **问题**：3 个测试失败
  - `tests/integration/api/test_jobs_api.py`：404 错误详情断言有乱码
  - `tests/integration/api/test_resume_api.py`：调用了已删除的路由 `/summary-history/`，断言了不存在的字段 `optimization_type`
- **决策**：修复所有 3 个测试断言，使其与当前 API 实现一致
- **理由**：测试基线必须与 API 实现保持一致
- **实现**：
  - `test_jobs_api.py`：`"岗位不存在"` 替换乱码
  - `test_resume_api.py`：路由 `/summary/history/` 替换 `/summary-history/`，字段 `mode` 替换 `optimization_type`

### D4.5: E2E Demo Chain 标准化
- **问题**：手动验证 E2E Demo 链路容易遗漏，且每次变更都需要重复验证
- **决策**：创建 `tests/e2e/test_demo_chain.py`，覆盖完整 14 步业务链路
- **理由**：自动化测试比手动验证更可靠、可回归
- **实现**：
  - 新文件：`tests/e2e/test_demo_chain.py`
  - 覆盖：Register → Login → /me → Resume → Summary → Job → Match → Interview → Tracker → Advice → Persist → History → /health → /ready
  - Mock AI services（AsyncMock）

### D4.6: Tracker 完整链路补全
- **问题**：Demo Chain 原先只覆盖到空白的 Tracker 列表，缺少创建申请 + 生成建议 + persist 的完整链路
- **决策**：在 Demo Chain 中补充完整 Tracker 流程：创建申请 → 生成建议 → persist → history
- **理由**：覆盖所有 4 个 AI 服务接口（resume summary、job match、interview questions、tracker advice）
- **实现**：`test_demo_chain.py` 步骤 9-12 补全
