# 事件响应运行手册

本文档用于在 IT 运维与开发团队面临系统故障时的快速响应与沟通流程。

## 1. 严重性等级
- SEV1：核心功能不可用，业务处于不可用状态。
- SEV2：核心功能受影响，性能显著下降或条件受限。
- SEV3：低优先级问题，影响较小但需记录与改进。

## 2. 常见故障及处理要点
- App 启动失败
  - 检查日志：docker compose logs -f app
  - 验证 DATABASE_URL 是否可用，数据库是否就绪
  - 检查迁移状态：alembic current / alembic stamp head
  - 如无异常，重新启动：docker compose restart app

- AI 接口返回 500
  - 检查 OPENAI_API_KEY 是否配置正确，以及外部提供商状态
  - 查看云服务配额/限流情况
  - 尝试切换到备用提供商（若已实现）

- 数据库连接错误
  - 检查 DATABASE_URL、数据库主机/端口是否正确
  - 验证数据库是否允许来自应用服务器的连接
  - 重新应用迁移：alembic upgrade head

- AI 接口高延迟
  - 检查提供商状态页
  - 评估是否触发限流、重试策略，以及超时设置

- 其他未预期错误
  - 记录发生时间、影响范围、受影响的用户、错误代码等
  - 设计外部通知（如 Slack/邮件）并触发告警

## 3. 上报与升级路径
- 报告人：指定的 on-call 工程师/团队
- 报告方式：在 incident channel 发起，记下时间、影响范围和初步诊断
- 紧急升级：将问题升级为 SEV1 并通知相关人员，启动应急流程

## 4. 事后复盘模板
 - 事件摘要：简要描述故障、发生时期、影响范围
 - 时间线：关键事件时间点列表
 - 根因分析：查明根本原因及诱因
 - 已执行的纠正措施：包含代码/配置变更
 - 验证结果：是否验证通过，监控是否回到正常
 - 改进点与行动项：预防类似故障的改进计划
 - 附件：日志、截图、告警截图等
